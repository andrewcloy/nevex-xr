# Naming Conventions

- use snake_case for all asset IDs and filenames
- keep family folders separate: `glyph_icons`, `tile_icons`, `panels`, `alerts`
- use semantic names instead of visual descriptions
- keep labels and words outside rendered art
- when the same concept exists in multiple families, keep the base name identical
- use manifests as the source of truth for category, priority, and intended placement
- preferred output examples:
  - `glyph_icons/power.png`
  - `tile_icons/power.png`
  - `panels/thermal_panel_shell.png`
