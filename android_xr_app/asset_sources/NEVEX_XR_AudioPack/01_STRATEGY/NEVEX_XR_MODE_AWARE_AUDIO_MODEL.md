# NEVEX XR Mode-Aware Audio Model

## Purpose

Operational modes do not create separate audio systems. They change emphasis, default enablement, and cue behavior while reusing the same audio architecture.

## Military / Threat

Audio posture:

- visual-first
- low-signature by default
- sparse but high-confidence alerts

Priorities:

- disciplined threat cues
- critical link and system warnings
- restrained navigation assist

Guidance:

- suppress chatty confirmations
- detection cues should be high-value only
- haptic escalation can be considered for future integration

## Hunting / Game

Audio posture:

- outdoor field friendly
- optional prey detection audio
- controlled reacquire behavior

Priorities:

- prey cues
- quick capture confirms
- trail-back and return cues

Guidance:

- avoid nuisance repetition
- optional audible prey cues should be user-governed

## Inspection / Search

Audio posture:

- subtle
- detail-oriented
- documentation friendly

Priorities:

- annotation and capture confirms
- breadcrumb and return-path cues
- restrained anomaly mark cues

Guidance:

- more supportive than urgent
- low fatigue is critical

## Navigation / Patrol

Audio posture:

- navigation becomes more prominent
- still restrained
- movement support over theatrics

Priorities:

- heading correction
- waypoint and route cues
- event mark confirmations
- deviation warnings

Guidance:

- keep waypoint and heading cues distinct
- avoid constant cadence

## Review / Playback

Audio posture:

- calm
- media-control oriented
- not scene-driven

Priorities:

- playback transport
- media selection
- protect, export, and delete confirmations

Guidance:

- use a calmer subset of UI sounds
- avoid operational alert tone character unless a fault occurs
