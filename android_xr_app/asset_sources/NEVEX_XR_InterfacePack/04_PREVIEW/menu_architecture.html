NEVEX XR Menu Architecture

Production-oriented XR hierarchy for a view-first advanced vision headset. The backbone stays stable, while operational modes change emphasis, HUD defaults, quick tools, alerts, capture behavior, and navigation support.

Layer 1

Live HUD
Battery, record state, current mode, zoom, and only contextual warnings.

Layer 2

Quick Menu
Brightness, gain/exposure, zoom, view mode, thermal blend, record, overlays, standby.

Layer 3

Main Menu
Vision, Thermal, Capture, Playback, Device, System, Advanced.

Layer 4

Advanced / Service
Calibration, diagnostics, service, recovery, developer tools.

Mode Model

Boot / Readiness Main mode select should route into mission behavior, not a second menu architecture.
Operational Modes Military / Threat, Hunting / Game, Inspection / Search, Navigation / Patrol, Review / Playback.
Shared Backbone Vision, Thermal, Capture, Playback, Device, System, Advanced remain stable in every mode.

Main Menu

Vision Image, enhancement, overlays, mode
Thermal Thermal view, palette, blend, hotspot
Capture Photo, video, defaults, metadata
Playback Recent, filter, actions
Device Power, storage, sensors, input and fit
System Connectivity, profiles, notifications, privacy and UI
Advanced Calibration, diagnostics, service, recovery, developer

Mode Emphasis

Military / Threat Emphasize Vision, Thermal, Capture, disciplined alerts, threat markers, bearing, quick evidence capture.
Hunting / Game Emphasize Vision, Thermal, Capture, prey cues, outdoor overlays, trail-back and camp return support.
Inspection / Search Emphasize detail, thermal anomaly assist, annotation, documentation, breadcrumb return logic.
Navigation / Patrol Emphasize heading, waypoint, route, event marking, obstacle-safe overlays, thermal assist as secondary.
Review / Playback Emphasize recent files, tagged events, route review, export, protect, and delete workflows.

Quick Menu Rules

Keep it to 6-8 items Quick controls should not become a mini settings dashboard.
Direct action first Prefer sliders, toggles, and segmented controls over deeper navigation.
Labels still matter Icons alone are acceptable in repeat-use quick controls, not in broad category navigation.
Mode-aware, not mode-chaotic Operational modes may swap one or two quick tools, but must not turn the quick menu into a mini settings app.

Navigation Integration

Not a separate front door Navigation is integrated mainly through Vision overlays, plus quick tools, mode tools, Playback review, and System preferences.
Cross-mode capability Threat uses bearing and return cues, hunting uses trail-back, inspection uses breadcrumbs, patrol uses route logic, playback reviews paths later.
Stay glanceable Use quiet heading strips, waypoint cues, and breadcrumb hints. Avoid map-heavy live overlays.

Icon Policy

Icons only HUD indicators, learned quick controls, contextual overlays.
Icons + labels Main menu categories, submenu rows, profiles, calibration, export, and advanced tools.
Deprioritized from top-level navigation Diagnostics, FPS, latency, target overlay states, and recovery indicators.
