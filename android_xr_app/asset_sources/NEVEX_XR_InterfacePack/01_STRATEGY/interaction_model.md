# Interaction Model

## View-first principles

The environment is the product. Every UI element competes with real-world visibility, so the default state must be quiet. Persistent UI should be limited to critical status, current mode, and a few high-frequency controls.

## HUD vs side panel vs full settings

HUD belongs to live-state essentials: current mode, target cues, capture, link, battery, and rare warnings. Side panels belong to quick adjustments that need fast repetition but not continuous presence. Full settings belongs to anything multi-step, explanatory, calibration-heavy, or rarely changed.

## One-click controls

One-click controls should include record, snapshot, IR toggle, detection toggle, mode switch, recenter, and quick settings entry. These actions must have clear visual feedback and should not require dense text.

## What must never clutter live view

Avoid persistent multi-row menus, heavy diagnostics, detailed settings labels, profile explanations, and ornamental tile grids inside the live mission view. These belong in panel or full settings layers only.

## XR-safe spacing and simplicity

Use larger-than-phone spacing, broad hit targets, and conservative element counts. Icons should read quickly in peripheral vision. Do not rely on tiny deltas between states; active, warning, and disabled states need strong contrast differences.
